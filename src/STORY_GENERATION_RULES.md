# 디지털 독서 인터랙티브 게임 제작 규칙

## 📌 프로젝트 개요
- **대상**: 초등학교 2학년 ~ 중학교 3학년
- **형식**: AI 기반 몰입형 인터랙티브 독서 게임
- **목적**: 고전소설을 2인칭 시점으로 체험하며 독서 흥미 유발
- **구조**: 프리퀄 형태 (작품의 본 이야기가 아닌, 세계관 체험)
- **장면 수**: **최소 30개 이상** (풍성하고 깊이 있는 스토리 전개)

---

## 📁 폴더 구조

```
/stories
  /elementary-2-3     # 초등학교 2~3학년 작품
  /elementary-4-6     # 초등학교 4~6학년 작품
  /middle-school      # 중학생 작품
/utils
  storyGenerator.ts           # 메인 스토리 생성기 (import만 사용)
  storyGeneratorKorean.ts     # 한국 고전 작품
  storyGeneratorMiddleSchool.ts # 중학생 작품
```

### 파일 생성 규칙
1. **새 작품 추가 시**: `/stories/[학년폴더]/작품명.ts` 파일 생성
2. **export 함수명 규칙**: `generate[작품명영문]Story`
   - 예: `generateTomSawyerStory`, `generateHongGilDongStory`
3. **index.ts에 export 추가**: 해당 학년 폴더의 index.ts에 추가
4. **utils 파일에 import 추가**: 학년별로 해당 utils 파일에 import
   - 초등 2~3학년: `/utils/storyGenerator.ts`
   - 초등 4~6학년: `/utils/storyGeneratorKorean.ts`
   - 중학생: `/utils/storyGeneratorMiddleSchool.ts`
5. **🚨 App.tsx에 작품 제목 추가 (필수!)**: 
   - **첫 화면 작품 선택 목록에 표시하려면 반드시 필요!**
   - `/App.tsx` 파일의 `booksByGrade` 객체에 작품명 추가
   - **이 단계를 빠뜨리면 작품이 선택 목록에 나타나지 않습니다!**

### 📋 새 작품 추가 체크리스트 (중요!)

**🚨 필수 4단계를 모두 완료해야 게임에서 선택 가능합니다!**

**만약 작품이 선택 목록에 안 보인다면?**
→ **4단계 (App.tsx의 booksByGrade)를 확인하세요!**

## 1. 기본 구조 규칙

### 1.1 장면 구성
- **장면 수**: **최소 30개 이상** (더 많을수록 좋음)
- **각 장면 요소**:
  - `id`: 고유 식별자
  - `title`: 장면 제목
  - `text`: 본문 (2인칭 시점, **8-12줄**)
  - `bg`: 배경 이미지 URL (Unsplash)
  - `choices`: 선택지 배열 (2개 권장)
  - `prompt`: 💭 질문 (한 줄로 간결하게)

### 1.2 스토리 형식
- **시점**: 2인칭 "너" 사용
- **구성**: **프리퀄 형태** (원작 스토리 X, 세계관/배경 체험)
- **목적**: 작품의 인물, 시대, 배경을 이해하고 흥미를 느끼게 함
- **몰입**: 학생이 주인공이 되는 경험 제공

### 1.2.1 페이지 제목 작성 규칙 (중요!)

**목적**: 각 페이지 제목에서 작품의 핵심 주제가 명확하게 드러나야 함

**필수 원칙**:
1. **주제 기반 제목** - 모든 페이지 제목은 작품의 핵심 주제를 직접적으로 표현
2. **구체적 가치 표현** - 추상적 제목보다 구체적 철학/신념/가치관 표현
3. **질문형 or 선언형** - "~란 무엇인가", "~의 의미", "~을 향하여" 등
4. **일반적 제목 지양** - "새로운 시작", "첫 만남" 같은 상황 설명 X

**❌ 나쁜 예시** (일반적, 주제 불명확):
- "선택의 시작" → 무슨 선택인지 불명확
- "첫 수련" → 상황 설명일 뿐
- "마을에 가다" → 사건 나열
- "도사님의 가르침" → 추상적
- "세월이 흐르다" → 주제 없음

**✅ 좋은 예시** (주제 명확):

**배움 주제:**
- "배움은 자유다"
- "지식과 지혜의 차이"
- "가르침의 책임"
- "책 속의 세상"
- "배움에는 끝이 없다"
- "실천 없는 지식"
- "스승의 마지막 가르침"

**애국 주제:**
- "나라란 무엇인가"
- "백성이 곧 나라"
- "평화를 위한 힘"
- "나라를 지킨다는 것"
- "진정한 충성"
- "전쟁과 평화"

**성차별 주제:**
- "차별받는 사람들"
- "평등을 향한 외침"
- "편견의 벽"
- "모두가 배울 권리"
- "두려움이 만든 차별"
- "여자도 영웅이 될 수 있다"
- "성별을 넘어서"

**정의 주제:**
- "옳고 그름의 경계"
- "정의란 무엇인가"
- "약자를 지키는 힘"
- "불의에 맞서다"

**효 주제:**
- "부모를 향한 사랑"
- "효의 진정한 의미"
- "희생과 사랑"

**작성 팁**:
1. 작품의 핵심 주제 3-4개를 먼저 정리
2. 각 장면이 어떤 주제를 다루는지 파악
3. 그 주제를 직접적으로 표현하는 제목 작성
4. "~의 의미", "~란 무엇인가", "~을 향하여" 같은 철학적 표현 활용

**예시 - 홍계월전 (배움, 애국, 성차별):**
- ❌ "선택의 시작" → ✅ "배움을 향한 첫걸음"
- ❌ "마을에 가다" → ✅ "차별받는 사람들"
- ❌ "무예 수련" → ✅ "힘의 진정한 의미"
- ❌ "책을 읽다" → ✅ "지식이 주는 자유"
- ❌ "도사님 말씀" → ✅ "나라란 무엇인가"

### 1.3 스토리 흐름 구조 (중요!)

**목적**: 무한루프 없이 모든 분기가 자연스럽게 엔딩으로 수렴하며, 선택마다 의미 있는 차이 제공

**필수 규칙**:
1. **엔딩은 하나** - 모든 선택지가 결국 같은 'ending'으로 연결
   - **🚨 엔딩 페이지 id는 반드시 `'ending'`으로 작성!**
   - **🚨 다른 장면에서 엔딩으로 연결할 때는 `to: 'ending'` 사용!**
2. **수렴 구조** - 여러 분기도 중반부에 자연스럽게 합류
3. **순환 금지** - 이전 장면으로 되돌아가는 선택지 금지
4. **깊이 제한** - start → ending까지 최대 15-20단계 이내
5. **반복 장면 금지** - 같은 장소/상황을 다른 id로 반복하지 않기
   - ❌ 나쁜 예: `go_to_attic_morning`, `go_to_attic_evening`, `go_to_attic_night` (다락방 반복)
   - ❌ 나쁜 예: `see_rabbit`, `meet_rabbit_again`, `rabbit_appears`, `find_rabbit` (토끼 반복)
   - ✅ 좋은 예: `go_to_garden` (한 번만), `meet_godfather` (새로운 캐릭터)
6. **의미 있는 선택** - 각 선택이 다른 경험/감정/결과를 제공해야 함
   - ❌ 나쁜 예: 선택 A → 장면1 → 장면2, 선택 B → 장면3 → 장면2 (결국 같은 곳)
   - ✅ 좋은 예: 선택 A → 용기를 내는 경험, 선택 B → 신중한 관찰의 경험 (다른 배움)
7. **선형 장면 최소화** - 선택지 없이 연속되는 장면은 하나로 통합
   - ❌ 나쁜 예: 장면A(선택 없음) → 장면B(선택 없음) → 장면C(선택 없음)
   - ✅ 좋은 예: 장면A에 모든 내 포함하고 바로 선택 제공

**구조 패턴**:
```
start (1개) - 2-3개 선택지
  ↓ (다양한 경험)
초반부 (3-5개 장면, 다양한 분기)
  ↓
허브1 (수렴) - 2-3개 선택지
  ↓ (다양한 경험)
중반부 (3-5개 장면, 다양한 분기)
  ↓
허브2 (수렴) - 2-3개 선택지
  ↓ (다양한 경험)
후반부 (3-5개 장면, 수렴)
  ↓
허브3 (최종 수렴)
  ↓
ending (1개)
```

**선택지 설계 원칙**:
1. **2-3개 선택지 제공** - 1개는 너무 선형적, 4개 이상은 복잡함
2. **각 선택의 고유한 가치**:
   - 선택 A: 적극적/외향적 행동 → 관계 형성 경험
   - 선택 B: 신중한/내향적 행동 → 자기 성찰 경험
   - 선택 C (선택적): 중립적 행동 → 관찰과 배움
3. **짧은 선형 구간 허용** - 2-3개 장면 후 다시 선택 제공
4. **허브 장면의 역할** - 모든 경로를 자연스럽게 모으는 중간 지점

**좋은 예시** - 의미 있는 분기:
```
start → (선택A: 말한다 / 선택B: 참는다)
  선택A → 거절당하지만 용기 배움 → 허브1
  선택B → 혼자지만 인내 배움 → 허브1
허브1 → (선택A: 탐험 / 선택B: 관찰)
  선택A → 새 친구 만남 → 허브2
  선택B → 자연과 교감 → 허브2
허브2 → (모든 경로 통합) → ending
```

**나쁜 예시** - 무의미한 선형 구:
```
start → 장면1(선택 없음) → 장면2(선택 없음) → 장면3(1개 선택만) → 장면4...
(선택지가 없고 그냥 읽기만 하는 구조)
```

**허브 장면 (수렴 지점) 설계**:
- 여러 경로가 만나는 중간 지점을 2-3개 만들기
- 예: `arrive_garden`, `meet_mentor`, `important_decision`
- 각 허브에서 다시 2-3개 선택지 제공
- 모든 분기가 최종적으로는 이 허브들을 거쳐 ending으

**반복 방지 전략**:
1. **같은 장소는 한 번만** - 다락방, 정원, 토끼 등 같은 소재를 여러 id로 나누지 말기
2. **새로운 캐릭터/사건 추가** - 반복 대신 대부님, 선생님, 친구 등 새로운 인물 등장
3. **시간 진행 명확히** - "며칠 후", "다음 날" 등으로 시간 흐름 표시하며 수렴

**체크리스트**:
- [ ] 모든 분기가 ending으로 연결되는가?
- [ ] 순환 구조가 없는가?
- [ ] 중간에 합류 지점(허브 장면)이 2-3개 있는가?
- [ ] start에서 ending까지 너무 길지 않은가? (15-20단계 이내)
- [ ] 같은 장소/상황을 반복하는 장면이 없는가?
- [ ] 다양한 캐릭터와 사건이 등장하는가?
- [ ] 각 선택이 다른 경험/감정을 제공하는가?
- [ ] 선택지 없이 연속되는 장면이 3개 이상 없는가?
- [ ] 대부분의 장면에서 2개 이상의 선택지를 제공하는가?

## 2. 텍스트 작성 규칙

### 2.1 기본 원칙
✅ **DO (이렇게 하세요)**
- 2인칭 "너" 시점 사용
- **대명사 우선 사용** (너는, 네가, 너를, 너와, 네)
- 한 줄로 간결한 💭 질문
- 연령대에 맞는 쉬운 어휘
- 괄호 안에 현대적 비유 설명 추가
- **성별 중립적 표현 사용** (남녀 모두 사용 능하게)

❌ **DON'T (피하세요)**
- ~~이름+조사 조합~~ (예: ~~${name.은는}~~, ~~${name.이가}~~, ~~${name.의}~~)
  - **❌ 나쁜 예**: "${name.이가} 있으면 재미있어!" 
  - **✅ 좋은 예**: "네가 있으면 재미있어!"
- ~~복잡하고 긴 질문~~
- ~~어려운 한자어나 고어~~
- ~~설명 없는 시대 배경~~
- **~~성별을 특정하는 표현~~** (아가씨, 도련님, 소년, 소녀, 양, 공자 등)

**이름 사용 규칙**:
- 시작 부분: `너의 이름은 ${name.full}.` ✅
- 대화서 호칭: `"${name.first}, 어서 와!"` ✅  
- 나머지는 **대명사 사용**: "너는", "네가", "너를", "네" ✅

### 2.1.1 성별 중립화 규칙 (중요!)

**목적**: 남학생과 여학생 모두 자연스럽게 몰입할 수 있도록 성별을 특정하지 않음

**❌ 절대 사용 금지 표현**:
- **호칭**: ~~"아가씨"~~, ~~"도련님"~~, ~~"소년"~~, ~~"소녀"~~, ~~"양"~~, ~~"공자"~~
- **명사**: ~~"사내아이"~~, ~~"계집애"~~, ~~"여자아이"~~, ~~"남자아이"~~
- **외모**: ~~"름다운"~~ (여성적), ~~"ermint한"~~ (남성적)
- **행동**: ~~"숙녀답게"~~, ~~"사나이답게"~~

**✅ 수정 방법**:

**1. 호칭 제거 또는 수정**
- ❌ "아가씨, 인형 놀이 하실래요?"
- ✅ "인형 놀이 하실래요?"

- ❌ "아가씨, 진정하세요!"
- ✅ "진정하세요!"

- ❌ "예, 아가씨."
- ✅ "예."

- ❌ "우리 아가씨가 착하네요!"
- ✅ "정말 착하네요!"

**2. 대명사로 대체**
- ❌ "아가씨는 소중해요."
- ✅ "소중해요." (주어 생략)
- ✅ "당신은 소중해요." (격식체 필요 시)

**3. 성별 중립적 명사 사용**
- ❌ "작은 소녀"
- ✅ "어린아이"

- ❌ "용감한 소년"
- ✅ "용감한 아이"

### 2.3 텍스트 길이 제한 (중요!)

**목적**: 모바일/웹 화면에서 스크롤 없이 선택 버튼이 보이도록 함

**필수 규칙**:
- **최소 8줄 ~ 최대 12줄** (빈 줄 포함)
- 너무 짧으면 내용이 부족하고, 13줄 이상이면 스크롤 발생
- 문단 간 빈 줄을 포함하여 계산
- 적절한 분량으로 몰입감 있는 스토리 전달
- **줄바꿈을 너무 자주 사용하지 말 것** - 주요 대사나 문단 나누기 위주로만 사용

**작성 가이드**:
1. **적절한 분량 유지**
   - ❌ 너무 짧음: "바다를 봤어. 좋았어." (2줄)
   - ❌ 너무 김: 13줄 이상의 긴 설명
   - ✅ 적절함: 8-12줄로 상황과 감정을 충분히 전달

2. **간결하지만 풍부하게**
   - 불필요한 수식어는 제거
   - 하지만 감정, 상황, 분위기는 충분히 묘사
   - 대화와 행동을 균형있게 배치
   - 한 문단 안에 여러 문장을 담아 자연스럽게 연결

3. **1~2글자 줄바꿈 방지**
   - 문장 길이를 조절하여 한 줄에 최소 10글자 이상
   - 어색한 줄바꿈이 생기면 문장을 재구성

**체크리스트**:
- [ ] 빈 줄 포함 8-12줄인가?
- [ ] 너무 간결하지 않고 충분한 내용이 있는가?
- [ ] 1~2글자만 줄바꿈되는 부분이 없는가?
- [ ] 핵심 내용과 감정이 잘 전달되는가?

### 2.4 텍스트 깨짐 방지

**문제**: UTF-8 인코딩에서 일부 한글이 깨지는 현상 (예: "")

**필수 규칙**:
1. **모든 한글 텍스트를 완전하게 작성**
   - 절대 중간에 글자가 잘리면 안 됨

   - "보물" ✅ / "보" ❌
   - "위험" ✅ / "위" ❌

   - "궁금" ✅ / "궁" ❌
   - "믿는" ✅ / "믿" ❌

2. **UTF-8 인코딩 유지**
   - 파일 저장 시 UTF-8 BOM 없이 저장
   - 이스케이프 시퀀스 사용 금지

3. **검증 방법**:
   - 작성 후 "" 패턴이 있는지 검색
   - 모든 한글 단어가 완전한지 확인

4. **최종 검수 (필수!)**:

   - 모든 텍스트가 한글로만 작성되었는지 확인

**체크리스트**:

### 3.3 선택지 작성법
- **2개 권장** (가끔 3개 가능)
- **능동적 행동** 제시
- **명확한 결과** 있는 선택
- **다양한 내용**: 각 장면의 선택지는 서로 다른 내용으로
- **뒤로 가기**: UI 레벨에서 항상 제공되므로, 스토리 선택지로는 특별히 필요하지 않음
  - 앱 상단에 항상 "뒤로" 버튼이 표시됨
  - 키보드 ← 키로도 뒤로 가기 가능
  - 첫 페이지에서는 자동으로 비활성화됨
  - 라서 "다시 생각해본다" 같은 선택지는 **특별한 경우에만** 사용

#### 일반 선택지 예시:
```javascript
choices: [
  { label: '바로 모험을 떠난다', to: 'adventure', cls: 'bg-red-500' },
  { label: '이모에게 허락받고 간다', to: 'ask_permission', cls: 'bg-blue-500' }
]
```

#### 뒤로 가기를 스토리 선택지로 포함하는 특별한 경우:
스토리 흐름상 "망설임"이나 "재고"가 중요한 의미를 가질 때만 사용:
```javascript
choices: [
  { label: '기를 내어 도전한다', to: 'brave_choice', cls: 'bg-red-500' },
  { label: '아직은 준비가 안 됐다', to: 'reconsider', cls: 'bg-gray-400' }
]
```

**뒤로 가기 선택지를 스토리에 넣을 때**:
- **라벨**: "아직은 아니야", "좀 더 고민한다", "다시 생각해본다" (단순히 "뒤로 가기"는 X)
- **색상**: `bg-gray-400` 또는 `bg-gray-500` (중립적 색상)
- **사용 시점**: 
  - 캐릭터의 망설임이 스토리상 의미가 있을 때
  - 중요한 선택의 무게감을 표현할 때
- **주의**: UI에 항상 뒤로 가기 버튼이 있으므로, 거의 필요 없음

### 4.6.1 추가 이미지 소스 활용법

**Pexels 검색**:
- URL 형식: `https://www.pexels.com/photo/...`
- Unsplash와 유사한 검색 방법
- 더 다양한 일러스트레이션 제공
- 특히 동화적 분위기 이미지에 유용

**Pixabay 검색**:
- URL 형식: `https://pixabay.com/photos/...`
- 일러스트, 벡터 아트도 풍부
- 클래식한 분위기의 예술 작품 많음
- 역사적 배경 이미지에 적합

**검색 전략**:
1. **1순위**: Unsplash 검색 (가장 고품질)
2. **2순위**: Pexels 검색 (일러스트 필요 시)
3. **3순위**: Pixabay 검색 (클래식 아트 필요 시)
4. **혼합 사용**: 같은 작품 내에서 다양한 소스 조합 가능

**예시 - 로빈슨 크루소 항구 장면**:
```typescript
// 실사 사진 원할 경우

// 페인팅 스타일 원할 경우

// 클래식 아트 원할 경
```

### 4.7 이미지 선택 프로세스 (2025-11-22 업데이트: 엄격한 기준 적용)

#### 📋 페이지별 이미지 검색 6단계

**1단계: 장면 내용 완전 분석**
- ✅ 어디서 일어나는가? (장소/배경)
- ✅ 무슨 일이 일어나는가? (사건/행동)
- ✅ 어떤 감정인가? (분위기/정서)
- ✅ 어떤 시대인가? (시대적 배경)
- ✅ 무엇이 핵심인가? (가장 중요한 요소)

**2단계: 핵심 키워드 정밀 추출**
- ✅ **장소 키워드**: 구체적인 공간 (예: 어두운 동굴, 고풍스러운 여관)
- ✅ **분위기 키워드**: 감정/정서 (예: 긴장감, 평화로운, 신비로운)
- ✅ **시대 키워드**: 역사적 시기 (예: medieval, traditional Korean, 1800s)
- ✅ **핵심 요소 키워드**: 결정적 사물/상징 (예: 마법 검, 한복, 범선)

**3단계: Unsplash 검색 (2-3개 키워드 조합)**
```typescript
// ✅ 좋은 예시 (구체적이고 적절한 조합)

unsplash_tool({ query: "dark mysterious cave" })          // 어두운 신비로운 동굴

unsplash_tool({ query: "equipment" })                     // 장비 (미용 장비 나올 수 있음)
unsplash_tool({ query: "house" })                         // 집 (현대 건물 나올 수 있음)
```

- ✅ **문화권 일치**: 작품 문화권과 완벽히 일치하는가?
- ✅ **내용 일치**: 페이지 내용을 정확히 반영하는가?

3. **choices 배열에 반드시 '처음부터 다시하기' 버튼 포함** ✅
4. **📚 책 읽기 권유 문구 포함** ✅
5. **8-12줄 규칙 엄수** ✅

```typescript
{
  id: 'ending',  // ← 필수! 다른 이름 사용 금지
  title: '프리퀄 에피소드 끝',  // ← 모든 작품 통일!
  text: `[프리퀄 요약 2-3줄]

[핵심 배움 정리]
- 배운 것 1
- 배운 것 2

[본편으로 연결]
그리고 오늘/이제...
[본편 시작점 설명]

---

📚 [본편 스토리 소개]

[작가명]의 《작품명》에서
[본편 핵심 내용 1-2줄]

[작품의 주제/가치 한 줄]

[THE END... 아니, THE BEGINNING!]`,
  bg: '[적절한 배경 이미지]',
  choices: [  // ← 필수! 빈 배열 절대 금지!
    { label: '🔄 처음부터 다시하기', to: 'start', cls: 'bg-색상-600' }
  ],
  prompt: '💭 《작품명》 - [작품 소개와 핵심 가치]'
}
```

**엔딩 작성 가이드**:
- **title**: '프리퀄 에피소드 끝' (모든 작품 통일)
- **text 마지막 줄**: '📚 서점이나 도서관에서 《작품명》을 찾아보자!' 필수
- **8-12줄 절대 엄수**: 스크롤 발생 방지!
- **간결화 전략**:
  - 불필요한 설명 제거
  - 긴 문장을 짧게 분리
  - "에는", "까지", "이야" 등 조사/어미 활용해 한 줄로 합치기
  - 작품 소개는 1-2줄로 압축
- **이모지 활용**: 📚 등으로 시각적 구분
- **prompt에 현대적 비유 필수**: 초등 2-3학년이 이해할 수 있는 게임/만화 비유

### 6.3 추천 멘트 작성법 및 스포일러 금지 (중요!)

**형식**: `《작품명》 - [최근 인기 콘텐츠 비유]! [한 줄 설명]`

**필수 규칙**:
- **비유 대상**: 웹소설, 웹툰, 드라마, 게임, 영화
- **톤**: 친근하고 흥미진진하게
- **연령대**: 각 학년에 맞는 비유 사용
- **🚨 스포일러 금지**: 원작의 구체적 결말이나 주요 사건을 밝히지 말 것!

**❌ 스포일러 예시 (절대 금지)**:
```
'《홍당무》 - 화가의 꿈을 이루는 이야기!' (결말 스포일러)
'《로미오와 줄리엣》 - 두 사람이 죽는 비극!' (결말 스포일러)
'《지킬박사와 하이드》 - 지킬이 하이드로 변신하는 이야기!' (핵심 반전 스포일러)
```

**✅ 좋은 예시 (스포일러 없음)**:
```
'《홍당무》 - 겨울왕국 안나처럼 차별받아도 자기만의 길을 찾는 이야기! 어려움 속에서 위로를 찾고 성장하는 감동 스토리야!'
'《로미오와 줄리엣》 - 가족이 반대하는 첫사랑 야! 진정한 사랑이 무엇인지 생각하게 해줘!'
'《지킬박사와 하이드》 - 마음속 어두운 면과 밝은 면의 이야기! 헐크처럼 감정 조절의 중요성을 배울 수 있어!'
```

**엔딩 텍스트 작성 시 주의사항**:
1. **결말 언급 금지** - "결국 ~한다", "마침내 ~를 이다" 같은 표현 피하기
2. **주제 분위기 위주** - 작품이 다루는 가치, 감정, 배경만 소개
3. **호기심 유발** - 독자가 "어떻게 될까?" 궁금하게 만들기
4. **과정 강조** - 결과보다는 "~하며 성장하는", "~를 배우는" 같은 과정 중심 서술

**초등 2~3학년 좋은 예시**:

## 7. 기술적 구현 규칙

### 7.1 이름 변수 사용
```javascript
const name = createNameVariations(lastName, firstName);

// 사용 가능한 변수:
name.full        // "김철수"
name.first       // "철수"
name.은는        // "철수는" (받침 자 처리)
name.이가        // "철수가" (받침 자동 처리)
name.을를        // "철수를" (받침 자동 처리)
name.의          // "철수의"
name.casual      // "철수야"
name.polite      // "김철수 학생"
```

**주의**: 텍스트에서는 대명("너", "네", "너는", "너를")를 우선 사용하고, 
정말 필요한 경우에만 `${name.full}` 사용

### 7.2 조사 처리
- 받침 유무에 따라 자동으로 은/는, 이/가, 을/를 선택
- `createNameVariations` 함수가 자동 처리

## 11. 독서 논술 대비 작품 제작 규칙 (장화홍련전 패턴)

### 11.1 핵심 주제 기반 설계

**목적**: 독서 논술 수업에서 다룰 주제를 프리퀄에서 미리 체험

**필수 단계**:
1. **작품의 핵심 주제 3-4개 정리**
   - 예시 (장화홍련전): 가족, 불안, 법
   - 예시 (홍길동전): 신분, 정의, 가족
   - 예시 (심청전): 효, 희생, 가족

2. **각 주제를 스토리에 자연스럽게 녹이기**
   - 주제마다 3-5개 장면 할당
   - 페이지 제목에서 주제 명확히 표현
   - 철학적 질문(prompt)으로 주제 강조

3. **페이지 제목 규칙**
   - ❌ "새로운 시작", "첫 만남" (일반적)
   - ✅ "가족의 부탁 - 첫 번째 약속" (주제 명확)
   - ✅ "법칙과 인간성 사이" (철학적)
   - ✅ "정의란 무엇인가" (가치관)

**장화홍련전 예시**:
```typescript
// 가족 주제
{ title: '가족의 부탁 - 첫 번째 약속', prompt: '💭 낯선 이의 부탁을 들어줘야 할 의무가 있을까?' }
{ title: '가족의 불안', prompt: '💭 가족은 서로를 지켜줘야 할 의무가 있을까?' }
{ title: '가족의 사랑 - 기억의 힘', prompt: '💭 사랑의 기억이 분노를 이길 수 있을까?' }

// 불안 주제
{ title: '보이지 않는 지켜보기', prompt: '💭 도울 수 없을 때 지켜보는 것은 의미가 있을까?' }
{ title: '무력함의 무게', prompt: '💭 규칙을 지키다가 누군가 다치면, 나는 책임이 없을까?' }

// 법 주제
{ title: '법칙과 인간성 사이', prompt: '💭 법과 정의가 충돌할 때, 무엇을 선택해야 할까?' }
{ title: '법이란 무엇인가', prompt: '💭 법이 완벽하지 않다면, 우리는 어떻게 해야 할까?' }
{ title: '진정한 정의란', prompt: '💭 진정한 정의는 처벌일까, 회복일까, 예방일까?' }
```

### 11.2 원작 스포일러 방지 전략

**원칙**: 프리퀄은 원작 본편 **시작점 직전**까지만 다룸

**방법**:
1. **원작 시작점 파악**
   - 장화홍련전: 홍련이 원님에게 나타나는 장면
   - 심청전: 심청이 인당수에 몸을 던지는 장면
   - 흥부전: 형제가 재산을 나누는 장면

2. **프리퀄 엔딩을 시작점 직전으로 설정**
   - 장화홍련전: "홍련이 원님을 찾아가는 모습을 지켜봤어"
   - 심청전: "심청이 배에 오르는 결심을 했어"
   - 흥부전: "형제가 서로 다른 길을 가기로 했어"

3. **원작 내용 최소화**
   - ✅ 배경과 세계관 설명
   - ✅ 인물의 성격과 가치관
   - ❌ 구체적 사건 전개
   - ❌ 결말이나 반전

### 11.3 철학적 질문 강화

**모든 prompt는 철학/신념/가치관 질문으로**

**질문 유형**:
1. **윤리/도덕**
   - "약속은 왜 지켜야 할까?"
   - "거짓말도 때로는 필요할까?"

2. **정의/법**
   - "법과 정의가 충돌할 때, 무엇을 선택해야 할까?"
   - "복수와 정의는 어떻게 다를까?"

3. **관계/가족**
   - "가족은 서로를 지켜줘야 할 의무가 있을까?"
   - "사랑하는 사람을 걱정하는 것도 고통이 될 수 있을까?"

4. **감정/심리**
   - "두려움을 느끼면서도 옳은 일을 하는 것, 그것이 용기!"
   - "평온함과 무관심은 어떻게 다를까?"

5. **선택/책임**
   - "중요한 선택을 할 때, 무엇을 기준으로 할까?"
   - "자유와 책임은 어떻게 균형을 이룰 수 있을까?"

### 11.4 선택지 버튼 색상 규칙 (파스텔 톤)

**필수 규칙**: 모든 선택지는 파스텔 톤 200 단계 사용

**색상 가이드**:
- `bg-purple-200`: 중요한 선택, 철학적 선택
- `bg-green-200`: 긍정적, 도움, 성장
- `bg-blue-200`: 신중함, 지혜, 관찰
- `bg-red-200`: 용기, 도전, 감정
- `bg-gray-200`: 중립, 망설임, 재고

**예시**:
```typescript
choices: [
  { label: '약속한다', to: 'promise', cls: 'bg-green-200' },
  { label: '더 자세히 묻는다', to: 'ask', cls: 'bg-blue-200' },
]

choices: [
  { label: '법칙을 지킨다', to: 'follow_law', cls: 'bg-blue-200' },
  { label: '작은 도움을 준다', to: 'help', cls: 'bg-green-200' },
]

choices: [
  { label: '복수를 선택한다', to: 'revenge', cls: 'bg-red-200' },
  { label: '용서를 선택한다', to: 'forgive', cls: 'bg-green-200' },
  { label: '정의에 맡긴다', to: 'justice', cls: 'bg-purple-200' },
]
```
