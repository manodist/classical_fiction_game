# bkit PDCA 기획서: 클래식 픽션 게임 프리미엄 개선 (Phase 1)
작성일: 2026-02-27
작성자: 에이전트 김비서 (Secretary Kim)
프로젝트: classical_fiction_game

## 1. 목적 (Goal)
초중등 대상 고전 소설 프리퀄 텍스트 게임의 기존 아키텍처(단일 엔딩, 성별 중립 규칙, 철저한 시대상 이미지)를 **100% 안전하게 유지**하면서, 시각적 우아함(UI/UX), 롤플레잉의 재미(Mechanics), 접근성과 희소성(Story & Content)을 순차적으로 결합하여 게임을 프리미엄 수준으로 업그레이드합니다.

## 2. 작업 원칙 (Core Principles)
1. **Zero-Breakage**: 현재 작동 중인 `STORY_GENERATION_RULES.md`의 구조와 `App.tsx`의 랜딩 루프를 절대 깨뜨리지 않음. 새로운 기능은 철저히 '선택적(Optional)'이거나 '비침투적(Non-invasive)' 레이어로 덧씌움.
2. **Sequential Rollout**: 에러 리스크를 차단하기 위해 [디자인] -> [재미] -> [스토리] 순으로 개발 단계를 완벽히 분리.
3. **PDCA 사이클 준수**: 각 단계 완료 후 bkit Gap Analysis를 통해 90% 이상의 달성도를 확인하고 다음 단계로 진입.

## 3. 기능별 기획 상세 (Plan Details)

### 3.1 🎨 디자인 (UI/UX) 개선
- **동적 텍스트 애니메이션**:
  - `framer-motion` 라이브러리를 추가 도입.
  - `GameScreen.tsx`의 텍스트 렌더링 부에 글자가 한 줄씩 나타나는 타이핑(Typewriter) 효과 적용.
- **동적 배경 (Ken Burns Effect)**:
  - 배경 이미지가 단순히 정지해 있는 것이 아니라 CSS/Framer Motion을 활용해 105%에서 100%로 30초에 걸쳐 매우 느리게 줌인/줌아웃되는 효과 추가.
- **사운드 피드백**:
  - 선택지 클릭, 다음 씬 전환 시 재생될 가벼운 효과음 파일(`assets/sfx/click.mp3` 등) 추가 및 `useAudio` 커스텀 훅 작성.
- **테마별 동적 틴트**:
  - 현재 배경색 혹은 게임 내 장면 상황(아침/밤 등)에 따라 버튼과 UI 테두리의 보색을 자동 계산하여 적용(CSS 변수 활용).

### 3.2 🕹️ 재미 요소 (Fun & Mechanics) 개선
- **선택지 성향 추가**:
  - 기존 `Type Scene`에 영향을 주지 않도록 데이터 레벨에서 선택지에 숨겨진 옵션 배정 (예: 용기+1, 지혜+1을 `data-affinity`로 처리).
  - 기존 스토리 정적 파일을 수정하지 않고 동적 계산 로직으로 성향 분석 후 엔딩 페이지 하단에 결과지 컴포넌트 부착.
- **수집품(Badge) 메커니즘**:
  - 특정 소설의 특정 씬에 방문 시 localStorage에 수집 ID 저장.
  - `MyRecordsPanel.tsx`에 '나의 고전 컬렉션' 탭 추가 및 획득한 배지 표시 로직 작성.

### 3.3 📖 스토리 및 콘텐츠 개선
- **TTS (읽어주기) 지원**:
  - 브라우저 내장 `window.speechSynthesis` (Web Speech API) 활용 (라이브러리 추가 불필요).
  - 우측 상단에 🔊 토글 버튼 추가. 한국어(ko-KR) 음성 엔진을 통해 현재 페이지 텍스트 낭독.
- **히든 쿠키 엔딩**:
  - 최종 수렴하는 `ending` id 씬에 도달했을 때, localStorage에 쌓인 성향 카운트가 특정 조건(All Perfect 등)을 만족하면 `<HiddenCookie/>` 컴포넌트 렌더링.

## 4. 리스크 관리 (Risk Management)
- **성능 저하 우려**: Framer Motion 의 잦은 애니메이션 작동이 저사양 태블릿에서 성능을 저하시킬 수 있음.
  - *대응책*: 유저 설정 혹은 저명확성 디바이스 감지 시 애니메이션을 끄는 옵션 배려 여부 검토.
- **기존 데이터 충돌 우려**: 스토리 파일(`*.ts`) 스키마 변경 시 과거 콘텐츠 하위 호환성 침해 가능성.
  - *대응책*: `Scene` 인터페이스 등을 변경할 때, 새로운 스탯 데이터 등은 모두 `optional (?)`로 선언하여 과거 파일 수정 소요를 0으로 만듦.
